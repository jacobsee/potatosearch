# Jacob See's Development Integrity Statement v1.0

This document is included in every project where nondeterministic tools have been used in any capacity — whether for composing code, creating tests, generating documentation, conducting reviews, or otherwise.

## Purpose

The presence of this file is an explicit acknowledgment that AI-assisted tooling has played a role in this project. It is a commitment to transparency.

## About the Author

I am a professional software engineer. I have produced high-quality code before the availability of coding assistants, and will I continue to do so regardless of the tooling landscape. AI/LLMs/assistants are not a substitute for understanding or judgment.

## Principles

- **Accountability**: Every change committed to this repository has been reviewed by a human.
- **Transparency**: This document exists so that collaborators, reviewers, and future maintainers are aware that AI tooling has been, in some capacity, part of the development process.
- **Quality**: The standard for code in this project is not "good enough for AI-generated." It is the standard that I believe to be appropriate for my own use of the software.

## Scope

This statement covers all forms of AI-assisted generation, including but not limited to:

- Source code
- Test cases
- Documentation
- Code reviews and suggestions
- Commit messages
- Configuration files

## Versioning

This is a generic document (v1.0) which I have created for reusability and is not customized per-project.
